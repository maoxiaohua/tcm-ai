## 🎯 核心架构

### 数据同步机制

```
左侧文本编辑
    ↓ (修改)
text_to_structured()
    ↓
structured_content (JSON) ← 数据库 (单一数据源)
    ↓
structured_to_text() + structured_to_tree()
    ↓
text_format + tree_structure
    ↓
刷新 → 左侧文本 + 右侧画布

右侧画布编辑
    ↓ (修改)
tree_to_structured()
    ↓
structured_content (JSON) ← 数据库 (单一数据源)
    ↓
structured_to_text() + structured_to_tree()
    ↓
text_format + tree_structure
    ↓
刷新 → 左侧文本 + 右侧画布
```

### 数据库保存策略

**保存时机**:
1. **左侧文本修改**: 500ms防抖后自动保存
2. **右侧画布修改**: 节点/连接变化后自动保存
3. **AI生成**: 生成后立即保存

**保存流程**:
```sql
-- 1. 保存主数据
INSERT/UPDATE clinical_decision_data SET
  structured_content = ?, -- JSON主数据
  text_format = ?,        -- 文本格式
  tree_structure = ?,     -- 树形结构
  last_modified_from = ?  -- 'text' 或 'canvas'

-- 2. 保存历史版本
INSERT INTO clinical_decision_history
  SELECT * FROM clinical_decision_data WHERE id = ?
```

# 按顿号、逗号、空格分割症状
parts = re.split(r'[、，,\s]+', symptoms_text)
