# 处方字段分离优化 - 修复说明

## 问题描述
用户反馈：在画布中编辑处方节点时，"处方名称"和"方剂组成"没有明确区分，容易混淆。

## 问题分析

### 原有设计
- 节点数据结构只有两个字段：
  - `node.name` → 节点名称
  - `node.description` → 详细描述
- 这个设计对于所有节点类型都是通用的

### 处方节点的映射
- `node.name` → **处方名称**（如："天麻钩藤饮"）
- `node.description` → **方剂组成**（如："天麻10g, 钩藤12g..."）

### 问题所在
画布节点编辑器的标签是通用的：
- "🏷️ 节点名称" → 不够明确
- "📝 详细描述" → 不够明确

对于处方类型节点，用户希望看到：
- "💊 处方名称" → 清楚表明这是处方名
- "💊 方剂组成" → 清楚表明这是药物成分

## 解决方案

### 修改内容
优化画布节点编辑器，根据节点类型动态显示不同的标签。

### 代码修改
**文件**: `/opt/tcm-ai/static/decision_tree_v3_data_driven.html`

**位置**: `editNode()` 函数中的HTML模板（第5430-5468行）

**修改前**:
```html
<label>🏷️ 节点名称</label>
<input placeholder="请输入节点名称">

<label>📝 详细描述</label>
<textarea placeholder="请输入详细的节点描述信息..."></textarea>
```

**修改后**:
```html
<label>
    ${node.type === 'prescription' ? '💊 处方名称' : '🏷️ 节点名称'}
</label>
<input placeholder="${node.type === 'prescription' ? '如：天麻钩藤饮' : '请输入节点名称'}">

<label>
    ${node.type === 'prescription' ? '💊 方剂组成' : '📝 详细描述'}
</label>
<textarea placeholder="${node.type === 'prescription' ? '如：天麻10g, 钩藤12g, 石决明15g...' : '请输入详细的节点描述信息...'}"></textarea>
```

## 数据流转说明

### 画布 → 结构化视图
1. 画布节点：`node.name = "天麻钩藤饮"`, `node.description = "天麻10g..."`
2. 构建结构化数据：
   ```javascript
   {
     id: "node_6",
     name: "天麻钩藤饮",           // 从 node.name
     ingredients: "天麻10g...",    // 从 node.description
   }
   ```
3. 渲染结构化视图：
   - **处方名称**: 天麻钩藤饮
   - **方剂组成**: 天麻10g...

### 结构化视图 → 画布
1. 结构化视图编辑：
   - 修改"处方名称" → `handleFieldEdit(element, nodeId, 'name')`
   - 修改"方剂组成" → `handleFieldEdit(element, nodeId, 'description')`
2. 更新画布节点：
   ```javascript
   node.name = newValue;        // 或
   node.description = newValue;
   ```
3. 画布自动重绘

### 画布编辑 → 结构化视图
1. 画布编辑器：
   - 修改"处方名称"输入框 → `node.name = newValue`
   - 修改"方剂组成"文本框 → `node.description = newValue`
2. 保存后：
   - 调用 `updateThinkingProcessFromNodes()`
   - 重新构建结构化数据
   - 重新渲染结构化视图

## 核心逻辑

### 1. 字段映射是正确的
- ✅ `node.name` → 处方名称
- ✅ `node.description` → 方剂组成
- ✅ 两者是独立的字段，不会混淆

### 2. 同步是正确的
- ✅ 画布修改 → 结构化视图自动更新
- ✅ 结构化视图修改 → 画布自动更新
- ✅ 修改哪个字段，就更新哪个字段

### 3. 问题不是数据混淆
- ❌ 不是"处方名称和方剂组成指向同一字段"
- ✅ 而是"标签不够清晰，用户容易搞混"

### 4. 解决方案
- ✅ 为处方节点显示专用标签
- ✅ 提供清晰的placeholder提示
- ✅ 保持数据结构和同步逻辑不变
