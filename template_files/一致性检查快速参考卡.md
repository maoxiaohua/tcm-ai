## 🎯 检查内容速查

| 检查项 | 优先级 | 说明 |
|-------|-------|------|
| **API接口一致性** | ⭐⭐⭐ | 前后端API是否匹配 |
| **参数命名一致性** | ⭐⭐⭐ | 参数名称是否统一 |
| **数据库字段引用** | ⭐⭐ | 字段是否存在于表中 |
| **数据结构一致性** | ⭐⭐ | 前后端数据格式是否匹配 |
| **常见错误模式** | ⭐⭐ | 已知错误是否重现 |
| **未使用API检测** | ⭐ | 发现冗余API |
| **数据库Schema** | ⭐ | 表结构完整性 |

| 分数 | 评级 | 说明 |

## 🔧 快速修复指南

### 1. API端点不存在

```
❌ 错误: /api/v2/auth/profile
调用文件: doctor/index.html
```

**修复**:
```javascript
// 方案1: 修改前端（如果端点错误）
fetch('/api/v2/auth/login')  // 使用正确端点

// 方案2: 添加后端路由（如果功能需要）
@router.get("/profile")
async def get_profile():
    ...
```

### 2. 参数命名不一致

```
⚠️ 警告: 使用 'selected_doctor', 建议 'doctor_id'
```

**修复**:
```javascript
// 方案1: 统一使用标准名称
{
  doctor_id: selectedDoctor  // 改为 doctor_id
}

// 方案2: 后端兼容两种格式
doctor_id = data.get('doctor_id') or data.get('selected_doctor')
```

### 3. 数据库字段不存在

```
❌ 错误: doctors.specialties (实际是 speciality)
```

**修复**:
```python
# 先检查表结构
# sqlite3 database.sqlite ".schema doctors"

# 使用正确字段名
cursor.execute("SELECT speciality FROM doctors WHERE id = ?", (id,))
```

### 4. 响应数据结构不一致

```
⚠️ 警告: 直接访问.reply可能失败
```

**修复**:
```javascript
// ❌ 错误
const reply = data.reply;

// ✅ 正确 - 兼容多种格式
let reply;
if (result.success && result.data) {
  reply = result.data.reply;
} else if (result.reply) {
  reply = result.reply;
} else {
  reply = '系统暂时无法响应';
}
```
