# 移除「更新思路」按钮 - 简化功能优化

## 📋 用户反馈

### 原始反馈
> "我不知道这个更新思路是干嘛用的，每次我点击就把左边的诊疗思路里面的内容格式改了，甚至变得不准确，实际上不需要这个什么更新思路"

### 核心问题
1. ❌ **功能不明确**：用户不清楚按钮的作用
2. ❌ **破坏性操作**：每次点击都会覆盖精心编辑的内容
3. ❌ **生成质量差**：自动生成的格式不如手动编辑准确
4. ❌ **无实际价值**：这个功能对用户的工作流程没有帮助

## 🎯 用户实际需求

### 期望的工作方式
```
左侧诊疗思路框           右侧画布
────────────────        ────────────────
完全手动编辑             可视化节点图
精确控制格式             辅助理清思路
不需要自动生成           独立存在
        ↓                      ↓
    两边完全独立，互不影响
```

### 真实使用场景
1. **诊疗思路**：医生直接在左侧输入框书写完整的临床思路
2. **画布节点**：作为辅助工具，帮助可视化疾病、证候、处方之间的关系
3. **独立保存**：两边内容独立保存，互不干扰

## ✅ 解决方案

### 移除按钮
直接移除画布工具栏中的"🔄 更新思路"按钮，避免用户误操作。

## 💡 设计理念

### 之前的设计（错误）
```
假设：用户希望画布节点自动生成诊疗思路
     ↓
提供"更新思路"按钮
     ↓
实际：用户不需要，反而造成困扰
```

### 现在的设计（正确）
```
理解：用户需要完全控制诊疗思路内容
     ↓
移除自动生成功能
     ↓
结果：两边完全独立，用户自由编辑
```

### 核心原则
1. **用户数据第一**：永远不要自动覆盖用户精心编辑的内容
2. **功能明确性**：每个按钮的作用应该清晰易懂
3. **简化界面**：移除不必要的功能，降低认知负担
4. **尊重习惯**：适应用户的实际工作流程，而不是强加假设
