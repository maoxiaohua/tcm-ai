# 诊疗思路更新策略优化

## 问题描述
用户反馈：创建或删除连接后，诊疗思路会自动更新，但这导致：
1. 手动编辑的格式被覆盖（详细格式变成简单格式）
2. 手动添加的内容丢失
3. 用户体验差

## 解决方案：从自动更新改为手动更新

### 核心策略
**不再自动更新诊疗思路，改为用户主动触发**

### 具体变更

#### 1. 创建连接时 - 不自动更新
**修改前**：
```javascript
connections.push(newConnection);
drawConnections();
updateThinkingProcessFromNodes();  // ❌ 自动更新
```

**修改后**：
```javascript
connections.push(newConnection);
drawConnections();
// ⚠️ 不自动更新，保留用户手动编辑的内容
console.log('💡 提示：连接已创建，但未自动更新诊疗思路（保留您的手动编辑内容）');
```

#### 2. 删除连接时 - 不自动更新
**修改前**：
```javascript
connections.splice(connectionIndex, 1);
drawConnections();
updateThinkingProcessFromNodes();  // ❌ 自动更新
```

**修改后**：
```javascript
connections.splice(connectionIndex, 1);
drawConnections();
// ⚠️ 不自动更新，保留用户手动编辑的内容
console.log('💡 提示：连接已删除，但未自动更新诊疗思路（保留您的手动编辑内容）');
```

#### 3. 新增"更新思路"按钮

**位置**: 画布右上角工具栏
**外观**: 🔄 更新思路（橙色按钮）
**功能**: 手动触发根据连接关系重新生成诊疗思路

**HTML代码**：
```html
<button id="updateThinkingBtn"
        class="canvas-tool-btn canvas-tool-btn-warning"
        title="根据连接关系重新生成诊疗思路"
        onclick="manualUpdateThinking()">
    🔄 更新思路
</button>
```

**JavaScript代码**：
```javascript
function manualUpdateThinking() {
    if (nodes.length === 0) {
        showResult('提示', '画布中没有节点', 'warning');
        return;
    }

    // 弹出确认对话框
    if (confirm('⚠️ 这将根据当前节点和连接关系重新生成诊疗思路。\n\n如果您手动编辑过诊疗思路，这些内容将被覆盖。\n\n是否继续？')) {
        console.log('🔄 用户手动触发：更新诊疗思路');
        updateThinkingProcessFromNodes();
        showResult('成功', '✅ 诊疗思路已根据连接关系重新生成', 'success');
    } else {
        console.log('❌ 用户取消了更新操作');
    }
}
```

**CSS样式**：
```css
.canvas-tool-btn-warning {
    background: linear-gradient(135deg, #f59e0b, #d97706);
    box-shadow: 0 2px 6px rgba(245, 158, 11, 0.3);
}

.canvas-tool-btn-warning:hover {
    background: linear-gradient(135deg, #d97706, #b45309);
    box-shadow: 0 4px 10px rgba(245, 158, 11, 0.4);
    transform: translateY(-2px);
}
```

## 使用流程

### 场景1：创建新的特色诊疗方案
1. 输入疾病名称
2. AI生成或手动添加节点
3. 手动创建连接关系
4. **点击"🔄 更新思路"按钮**
5. 确认后自动生成诊疗思路
6. 保存方案

### 场景2：编辑已有的特色诊疗方案
1. 加载已保存的方案（保留原有诊疗思路）
2. 添加/删除节点和连接
3. **诊疗思路不会自动改变**（保护手动编辑）
4. 如需重新生成：点击"🔄 更新思路"按钮
5. 确认后覆盖旧内容
6. 保存更新

### 场景3：只修改连接，不修改思路
1. 在画布上调整连接关系
2. **不点击"🔄 更新思路"按钮**
3. 保持左侧诊疗思路不变
4. 直接保存

## 对比：修改前后

### 修改前（自动更新）
❌ 创建连接 → 自动覆盖诊疗思路 → 手动编辑的格式丢失
❌ 删除连接 → 自动覆盖诊疗思路 → 用户困惑
❌ 无法选择保留原格式

### 修改后（手动更新）
✅ 创建连接 → 仅添加连接线 → 诊疗思路保持不变
✅ 删除连接 → 仅删除连接线 → 诊疗思路保持不变
✅ 需要更新时 → 点击"更新思路"按钮 → 确认后才更新
✅ 用户完全掌控何时更新

## 优势

### 1. 保护用户内容
✅ 手动编辑的详细内容不会丢失
✅ 自定义的格式和说明得以保留
✅ 避免意外覆盖

### 2. 灵活性
✅ 用户可以选择何时更新
✅ 可以在不影响思路的情况下调整连接
✅ 支持渐进式编辑

### 3. 透明性
✅ 明确提示是否会覆盖内容
✅ 需要确认才执行覆盖操作
✅ 用户完全掌控

### 4. 兼容性
✅ 兼容旧的自动生成功能（通过按钮触发）
✅ 不影响现有的节点编辑和保存功能
✅ 向后兼容已保存的方案

## 注意事项

### 何时需要点击"更新思路"按钮？

**需要点击**：
- ✅ 创建了很多连接，想根据连接关系生成思路
- ✅ 删除了连接，想同步更新思路
- ✅ 重新组织了节点关系，想重新生成
- ✅ 想用自动生成的简洁格式替换手动编辑

**不需要点击**：
- ❌ 只是微调连接，思路已经很完善
- ❌ 手动编辑了详细的诊疗思路
- ❌ 只是调整节点位置，不改变内容
- ❌ 保存前（连接关系会随方案一起保存）
