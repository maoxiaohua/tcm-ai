# 决策树数据驱动架构设计方案

## 📋 需求分析

### 您的核心需求

1. **标准格式** ⭐
   - 左侧诊疗思路和右侧画布内容格式一致
   - 有统一的数据结构标准

2. **智能生成** 🤖
   - 医生输入自然语言
   - AI智能生成标准格式的内容和画布

3. **实时同步** 🔄
   - 修改左侧 → 更新右侧
   - 修改右侧 → 更新左侧
   - 数据库作为唯一数据源

4. **数据库驱动** 💾
   - 所有内容存储在数据库
   - 左侧和右侧都是展示层
   - 易于扩展其他功能

## 🎯 设计理念

### 核心架构：**单一数据源（Single Source of Truth）**

```
┌─────────────────────────────────────────┐
│           数据库（唯一数据源）            │
│  - 疾病定义                              │
│  - 主证、症见                            │
│  - 处方名称、方剂内容                     │
│  - 诊疗思路、决策逻辑                     │
└──────────────┬──────────────────────────┘
               │
       ┌───────┴───────┐
       ▼               ▼
  ┌────────┐      ┌────────┐
  │左侧文本│◄────►│右侧画布│
  │（展示）│ 实时  │（展示）│
  └────────┘ 同步  └────────┘
```

**关键点**:
- ✅ 数据库是**唯一的真实数据**
- ✅ 左侧和右侧只是**不同的展示形式**
- ✅ 修改任何一侧，都写入数据库，然后刷新两侧

### 1. 标准数据结构

#### 核心数据表：`clinical_decision_data`

```sql
CREATE TABLE clinical_decision_data (
    -- 基础信息
    id TEXT PRIMARY KEY,
    doctor_id TEXT NOT NULL,
    disease_id TEXT NOT NULL,
    disease_name TEXT NOT NULL,

    -- 标准化内容（JSON格式）
    structured_content TEXT NOT NULL,  -- JSON: 结构化的诊疗内容

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- 审计
    last_modified_from TEXT,           -- 'text' 或 'canvas'
    modification_count INTEGER DEFAULT 0
);
```

## 🔄 同步机制设计

### 1. 数据流向

```
用户操作 → 数据库更新 → 自动刷新两侧展示

具体流程：
1. 用户修改左侧文本
   ↓
2. 解析文本 → 生成structured_content JSON
   ↓
3. 保存到数据库（更新text_format和structured_content）
   ↓
4. 从structured_content生成tree_structure
   ↓
5. 刷新右侧画布

或者：

1. 用户修改右侧节点
   ↓
2. 更新tree_structure JSON
   ↓
3. 保存到数据库（更新tree_structure和structured_content）
   ↓
4. 从structured_content生成text_format
   ↓
5. 刷新左侧文本
```

### 2. API设计

#### 核心API端点

```python
# 1. 智能生成（AI驱动）
POST /api/decision-tree/generate
{
  "doctor_id": "doctor_001",
  "natural_language": "风寒感冒，患者恶寒发热重...",
  "auto_standardize": true
}
→ 返回：标准化的structured_content

# 2. 保存/更新（任意一侧修改）
POST /api/decision-tree/save
{
  "id": "decision_001",
  "modified_from": "text" | "canvas",
  "content": {
    // 根据modified_from决定是text_format还是tree_structure
  }
}
→ 自动同步到另一侧

# 3. 获取数据（用于展示）
GET /api/decision-tree/{id}
→ 返回：完整的数据（包含text_format和tree_structure）

# 4. 实时同步（WebSocket，可选）
WS /api/decision-tree/sync/{id}
→ 实时推送更新
```

```python
    def structured_to_tree(self, structured_data: dict) -> dict:

        # 生成节点和连接
        pass

    def tree_to_structured(self, tree_data: dict) -> dict:
        """树形结构 → 结构化JSON"""
        # 从节点和连接提取信息
        pass

    def save_and_sync(self, id: str, content: any, source: str):
        """保存并同步

        Args:
            id: 数据ID
            content: 修改的内容
            source: 'text' 或 'canvas'
        """
        # 1. 根据source确定输入格式
        if source == 'text':
            structured = self.text_to_structured(content)
            tree = self.structured_to_tree(structured)
            text = content
        else:  # canvas
            structured = self.tree_to_structured(content)
            text = self.structured_to_text(structured)
            tree = content

        # 2. 保存到数据库
        self.save_to_database(id, structured, text, tree, source)

        # 3. 返回完整数据（用于刷新两侧）
        return {
            'structured_content': structured,
            'text_format': text,
            'tree_structure': tree
        }
```

#### 2.2 API路由

```python
# api/routes/decision_tree_data_routes.py

from fastapi import APIRouter, HTTPException
from services.decision_tree_data_service import DecisionTreeDataService

router = APIRouter(prefix="/api/decision-tree-data")
service = DecisionTreeDataService()

@router.post("/generate")
async def generate_from_natural_language(data: dict):
    """AI智能生成"""
    natural_text = data.get('natural_language')
    doctor_id = data.get('doctor_id')

    # 1. AI解析自然语言
    structured = await service.ai_parse_natural_language(natural_text)

    # 2. 生成两种格式
    text = service.structured_to_text(structured)
    tree = service.structured_to_tree(structured)

    # 3. 保存到数据库
    id = service.create_new(doctor_id, structured, text, tree)

    return {
        'success': True,
        'data': {
            'id': id,
            'structured_content': structured,
            'text_format': text,
            'tree_structure': tree
        }
    }

@router.post("/save")
async def save_decision_data(data: dict):
    """保存并同步"""
    id = data.get('id')
    content = data.get('content')
    source = data.get('modified_from')  # 'text' 或 'canvas'

    # 保存并同步
    result = service.save_and_sync(id, content, source)

    return {
        'success': True,
        'data': result,
        'message': f'已从{source}同步更新'
    }

@router.get("/{id}")
async def get_decision_data(id: str):
    """获取完整数据"""
    data = service.get_by_id(id)

    if not data:
        raise HTTPException(status_code=404, detail="数据不存在")

    return {
        'success': True,
        'data': data
    }

@router.post("/{id}/validate")
async def validate_consistency(id: str):
    """验证一致性"""
    data = service.get_by_id(id)

    # 检查三种格式的一致性
    issues = service.check_consistency(
        data['structured_content'],
        data['text_format'],
        data['tree_structure']
    )

    return {
        'success': True,
        'is_consistent': len(issues) == 0,
        'issues': issues
    }
```
