# 特色诊疗方案多格式导出功能使用指南

## 功能概述

现在支持将特色诊疗方案导出为 **3种专业格式**：
- 📄 **TXT文本** - 纯文本格式，结构清晰，易于阅读
- 📘 **Word文档** - HTML格式的.doc文件，支持格式化和表格
- 📝 **Markdown** - 开发者友好格式，支持代码块和表格

## 使用方法

### 1. 打开导出菜单
访问 https://mxh0510.cn/static/decision_tree_visual_builder.html

点击左侧的 **"我的特色方案"** 按钮，在弹出的列表中：

```
┌─────────────────────────────────────────────────┐
│ 头痛                            [导出 ▼] [删除] │
│ 暂无诊疗思路描述                                  │
│ 创建时间：2025/10/27 10:26:00 | 节点数：12       │
└─────────────────────────────────────────────────┘
```

### 2. 选择导出格式
点击 **"导出 ▼"** 按钮，会弹出格式选择菜单：

```
┌─────────────────┐
│ 📄 TXT文本      │
│ 📘 Word文档     │
│ 📝 Markdown     │
└─────────────────┘
```

### 3. 下载文件
选择格式后，文件会自动下载到本地。

文件命名格式：`特色诊疗方案_[疾病名称]_[时间戳].[扩展名]`

例如：`特色诊疗方案_头痛_1730012345678.txt`

## 导出内容结构

所有格式都包含以下完整信息：

### ✅ 基本信息
- 方案名称
- 创建时间
- 导出时间
- 节点数量

### ✅ 一、诊疗思路
完整的医生诊疗思维过程描述

### ✅ 二、决策树结构
1. **节点详情表格**
   - 序号
   - 节点内容
   - 位置坐标
   - 节点类型

2. **连接关系表格**
   - 序号
   - 起始节点
   - 目标节点
   - 连接标签

### ✅ 三、临床模式分析
结构化的临床诊疗模式数据

### ✅ 四、医生专业知识
医生专业领域知识和经验

### ✅ 导出说明
系统信息和导出工具说明

## 格式详细对比

### 📄 TXT文本格式
**适用场景**：打印、归档、纯文本查看

**特点**：
- 使用 `═` 和 `─` 字符绘制分隔线
- 层次清晰，易于阅读
- 兼容所有文本编辑器
- 文件体积最小

**示例结构**：
```
════════════════════════════════════════════════════════════
            特色诊疗方案
════════════════════════════════════════════════════════════

【方案名称】头痛

【创建时间】2025/10/27 10:26:00

【导出时间】2025/10/27 13:33:00

════════════════════════════════════════════════════════════
            一、诊疗思路
════════════════════════════════════════════════════════════

患者主诉头痛，需要详细询问...

────────────────────────────────────────────────────────────
【节点详情】
────────────────────────────────────────────────────────────

1. 症状询问
   位置：(100, 50)
   类型：起始节点

2. 诊断分析
   位置：(250, 50)
   类型：决策节点
...
```

### 📘 Word文档格式
**适用场景**：正式报告、演示文档、打印分发

**特点**：
- 完整的HTML格式化
- 包含样式、颜色、表格
- Microsoft Word可直接打开
- 支持编辑和二次排版

**样式特点**：
- 标题使用蓝色系配色
- 表格边框清晰
- 内容分区有背景色
- 字体：微软雅黑/宋体

**文件类型**：`.doc` (HTML格式)

### 📝 Markdown格式
**适用场景**：技术文档、版本控制、在线发布

**特点**：
- GitHub风格Markdown
- 支持表格和代码块
- 适合Git版本管理
- 可转换为HTML/PDF

**示例结构**：
```markdown
# 📋 特色诊疗方案：头痛

## 基本信息

| 项目 | 内容 |
|------|------|
| 创建时间 | 2025/10/27 10:26:00 |
| 导出时间 | 2025/10/27 13:33:00 |
| 节点数量 | 12 |

---

## 一、诊疗思路

患者主诉头痛，需要详细询问...

---

## 二、决策树结构

### 2.1 节点信息

| 序号 | 节点内容 | 位置 | 类型 |
|------|----------|------|------|
| 1 | 症状询问 | (100, 50) | 起始节点 |
| 2 | 诊断分析 | (250, 50) | 决策节点 |
...
```

## 使用建议

### 📄 选择TXT格式，当你需要：
- ✅ 快速查看内容
- ✅ 在任何设备上打开
- ✅ 纯文本存档
- ✅ 最小文件体积

### 📘 选择Word格式，当你需要：
- ✅ 正式医疗报告

- ✅ 美观的格式化呈现
- ✅ 进一步编辑排版

### 📝 选择Markdown格式，当你需要：

- ✅ 发布到Wiki/Blog
- ✅ 转换为其他格式

## 技术细节

### 文件编码
所有格式均使用 **UTF-8** 编码，完美支持中文

### 文件大小
- TXT: 约2-5KB（取决于内容长度）
- Word: 约5-10KB（包含格式化信息）
- Markdown: 约2-5KB（接近TXT大小）

### 浏览器兼容性
✅ Chrome/Edge (推荐)
✅ Firefox
✅ Safari
⚠️ IE11 (可能有兼容性问题)

## 常见问题

### Q: Word文档能在WPS中打开吗？
A: 可以。导出的.doc文件是HTML格式，WPS、Microsoft Word、LibreOffice都能打开。

### Q: 导出失败怎么办？
A: 检查以下几点：
1. 是否已登录
2. 浏览器是否允许下载
3. 查看浏览器控制台错误信息

### Q: 能批量导出所有方案吗？
A: 当前版本仅支持单个导出。批量导出功能正在开发中。

### Q: Markdown文件如何预览？
A: 推荐工具：
- VSCode + Markdown Preview
- Typora
- 在线工具：GitHub、GitLab、MarkdownPad

### Q: 导出后能重新导入吗？
A: 当前版本不支持导入。建议保留原始数据在系统中。
