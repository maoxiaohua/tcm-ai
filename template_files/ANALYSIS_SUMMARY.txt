💡 架构建议
================================================================================

当前架构: 高耦合 + 全局变量污染
推荐架构: 事件驱动 + 模块化 + 依赖注入

推荐结构:
/js
├── constants/          # 常量定义 (doctors, messages, errors等)
├── services/           # 业务服务 (consultation, doctor, prescription)
├── stores/             # 状态管理 (user, conversation, doctor)
├── ui/                 # UI组件 (chat, doctor, layout)
├── utils/              # 工具函数 (storage, format, validate)
├── event-bus.js        # 中央事件总线
└── app.js              # 应用主入口

优势:
✓ 低耦合: 模块通过事件总线通信
✓ 易测试: 每个模块独立可测
✓ 易扩展: 新功能无需修改现有代码
✓ 易维护: 职责明确便于定位问题
