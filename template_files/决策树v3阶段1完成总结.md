## 🎯 核心特性

### 1. 数据分离 ✅

```
原版本数据:                      新版本数据:
doctor_clinical_patterns     →   clinical_decision_data
(保持不变)                       (全新表结构)
```

**优势**:
- ✅ 测试数据不影响生产
- ✅ 原功能不受影响
- ✅ 可以安全测试

### 2. 标准格式 ✅

所有数据用**统一的JSON结构**存储：

```json
{
  "disease": {...},          // 疾病信息
  "diagnosis": {             // 诊断信息
    "main_symptoms": [...],  // 主症
    "secondary_symptoms": [...],  // 兼症
    "tongue": {...},         // 舌象
    "pulse": {...}           // 脉象
  },
  "syndrome": {...},         // 证候
  "prescriptions": [{        // 处方
    "name": "麻黄汤",
    "composition": [{        // 方剂组成
      "name": "麻黄",
      "dosage": 9,
      "unit": "克",
      "role": "君药"
    }]
  }],
  "decision_flow": {         // 决策流程
    "nodes": [...],
    "connections": [...]
  }
}
```

### 3. 自动同步设计 ✅

```
修改左侧文本
      ↓
保存到数据库（structured_content）
      ↓
生成右侧树形结构
      ↓
刷新右侧画布

（反向同理）
```
