# 连接同步节点类型修复

## 🐛 问题描述

### 用户反馈
用户在画布上创建连接线后（如从"脑袋瓜前面疼"连接到"处方2"），发现左侧诊疗思路没有自动更新处方关联关系。

### 问题场景
```
步骤1: 画布上有证候节点"脑袋瓜前面疼"和处方节点"处方2"
步骤2: 用户创建连接线：脑袋瓜前面疼 → 处方2
步骤3: 检查左侧诊疗思路
     ❌ 该证候的"处方："字段没有更新为"处方2"
     ❌ 连接关系同步失败
```

### 控制台日志
```
🔗 开始同步连接关系
  从: 脑袋瓜前面疼 (syndrome_branch)  ← 节点类型是 syndrome_branch
  到: 处方2 (prescription)
  操作: 创建连接
💡 未找到匹配的证候-处方关系，可能需要手动更新
```

## 🔍 根本原因分析

### 代码逻辑
```javascript
// 第2336行：syncConnectionToThinking() 函数
if (fromNode.type === 'syndrome' && toNode.type === 'prescription') {
    // 同步逻辑
}
```

### 问题根源
**类型匹配失败**：
- 代码检查：`fromNode.type === 'syndrome'`
- 实际类型：`fromNode.type === 'syndrome_branch'`
- 结果：条件判断失败，同步逻辑被跳过

### 为什么会有不同类型？
在决策树构建器中，证候相关的节点可能有多种类型：
- `'syndrome'` - 标准证候节点
- `'syndrome_branch'` - 证候分支节点
- 可能还有其他变体

但原代码只检查了 `'syndrome'` 一种类型，导致 `'syndrome_branch'` 类型的节点创建连接时同步失败。

## ✅ 解决方案

### 修改位置
文件：`/opt/tcm-ai/static/decision_tree_visual_builder.html`
行号：第2336行

### 修改前
```javascript
// 场景：证候 → 处方 的连接
if (fromNode.type === 'syndrome' && toNode.type === 'prescription') {
    console.log('📝 检测到证候-处方连接，尝试更新...');
    // ...
}
```

### 修改后
```javascript
// 场景：证候 → 处方 的连接（支持 syndrome 和 syndrome_branch 类型）
if ((fromNode.type === 'syndrome' || fromNode.type === 'syndrome_branch') && toNode.type === 'prescription') {
    console.log('📝 检测到证候-处方连接，尝试更新...');
    // ...
}
```

### 修改说明
使用逻辑OR运算符 `||` 同时检查两种证候节点类型：
- ✅ `'syndrome'` - 标准证候节点
- ✅ `'syndrome_branch'` - 证候分支节点

这样确保无论用户使用哪种证候节点类型，创建连接时都能正确触发同步逻辑。

## 🔧 技术细节

### 同步逻辑流程
```
1. 用户在画布上创建连接（或删除连接）
   ↓
2. 触发 syncConnectionToThinking(fromNode, toNode, action)
   ↓
3. 检查节点类型：
   - 起点：syndrome 或 syndrome_branch ✅
   - 终点：prescription ✅
   ↓
4. 构建正则表达式，匹配诊疗思路中对应证候的处方行
   ↓
5. 执行替换：
   - action='add': 更新处方名称
   - action='delete': 清空处方名称（保留标签）
   ↓
6. 应用更新到诊疗思路输入框
   ↓
7. 显示绿色边框视觉反馈（1秒）
```

### 正则表达式详解
```javascript
const syndromePattern = new RegExp(
    `(【证候\\d+】${escapeRegExp(fromNode.name)}[\\s\\S]*?处方[：:]\\s*)([^\\n]*)(\\n)`,
    'g'
);
```

**匹配逻辑**：
1. `【证候\\d+】` - 匹配证候标题（如"【证候1】"）
2. `${escapeRegExp(fromNode.name)}` - 匹配证候名称（转义特殊字符）
3. `[\\s\\S]*?` - 非贪婪匹配任意内容（包括换行）
4. `处方[：:]\\s*` - 匹配"处方："或"处方："标签
5. `([^\\n]*)` - 捕获组$2：当前处方名称（待替换部分）
6. `(\\n)` - 捕获组$3：保留换行符

**替换策略**：
- 添加连接：`$1${toNode.name}$3` - 保留前缀，替换处方名，保留换行
- 删除连接：`$1$3` - 保留前缀，清空处方名，保留换行
