## 🔍 检查内容

### 1️⃣ 参数命名一致性 ⭐⭐⭐

**检查内容**: 确保所有地方使用相同的参数名称

**检查结果示例**:
```
❌ 错误: 参数 'doctor_id' 使用了 5 种不同的命名

当前使用情况:
  - doctor_id: 76 次使用 ✓ 标准
  - doctor_name: 68 次使用 ✗ 别名
  - selected_doctor: 16 次使用 ✗ 别名
  - doctorId: 13 次使用 ✗ 别名
  - doctor_code: 4 次使用 ✗ 别名

建议: 统一使用 'doctor_id'
```

**权威标准参数表**:

| 标准名称 | 类型 | 格式 | 示例 | ❌ 错误别名 |
|---------|------|------|------|-----------|
| `user_id` | string | 用户ID | `user_12345` | userId, patient_id, patientId, uid |
| `doctor_id` | string | 拼音名称 | `zhang_zhongjing` | doctorId, selected_doctor, doctor_name |
| `conversation_id` | string | 会话ID | `conv_12345` | conversationId, session_id, sessionId |
| `prescription_id` | string | 处方ID | `prescription_12345` | prescriptionId, rx_id |
| `message` | string | 消息 | `我头痛三天了` | content, msg, text, body |

### 2️⃣ 医生ID格式检查 ⭐⭐⭐ (重点!)

**检查内容**: 确保医生ID使用**名称格式**而非数字

**常见错误**:
```javascript
// ❌ 错误 - 使用数字格式
{
  doctor_id: '1',  // 数字！
  selected_doctor: '2'
}

// ✅ 正确 - 使用名称格式
{
  doctor_id: 'zhang_zhongjing',  // 拼音名称
  selected_doctor: 'ye_tianshi'
}
```

**检测模式**:
- 赋值语句中的数字: `doctor_id: '1'`
- 映射表定义中的数字: `{'1': 'zhang_zhongjing'}`

**修复建议**:
```javascript
// 步骤1: 定义医生ID映射表（如果需要兼容旧代码）
const DOCTOR_ID_MAP = {
  '1': 'zhang_zhongjing',
  '2': 'ye_tianshi',
  '3': 'li_dongyuan',
  '4': 'zheng_qinan',
  '5': 'liu_duzhou'
};

// 步骤2: 在发送API请求前转换
const requestData = {
  message: userMessage,
  user_id: currentUser.id,
  doctor_id: DOCTOR_ID_MAP[selectedDoctorId] || 'zhang_zhongjing'
};

// 步骤3: 后端统一使用名称格式
# 后端不应该接受数字格式的医生ID
```

### 3️⃣ 响应数据访问一致性 ⭐⭐

**检查内容**: 确保安全访问API响应数据

**常见错误**:
```javascript
// ❌ 错误 - 直接访问可能不存在的属性
const reply = data.reply;  // 如果data没有reply属性会报错

// ✅ 正确 - 使用安全访问
const reply = data?.reply || result?.data?.reply || '默认值';

// ✅ 更好 - 使用统一的响应处理函数
const reply = extractReply(response);
```

**推荐方案**:
```javascript
// 创建统一的响应处理器
function extractReply(response) {
  // 处理新API格式 (统一问诊服务)
  if (response.success && response.data) {
    return response.data.reply || response.data.message;
  }

  // 处理旧API格式
  if (response.reply) {
    return response.reply;
  }

  // 处理直接返回
  if (typeof response === 'string') {
    return response;
  }

  // 默认值
  return '系统暂时无法响应，请稍后重试';
}
```

## 💡 开发规范（必须遵守）

### ✅ DO（必须这样做）

1. **使用标准参数名**
   ```javascript
   // ✅ 正确
   { user_id, doctor_id, conversation_id, prescription_id, message }
   ```

2. **医生ID使用名称格式**
   ```javascript
   // ✅ 正确
   doctor_id: 'zhang_zhongjing'
   ```

3. **安全访问响应数据**
   ```javascript
   // ✅ 正确
   const reply = response?.data?.reply || '默认值';
   ```

4. **修改代码前先运行检查**
   ```bash
   python3 scripts/parameter_consistency_enforcer.py
   ```

### ❌ DON'T（绝对不要这样做）

1. **不要使用参数别名**
   ```javascript
   // ❌ 错误
   { patient_id, selected_doctor, sessionId, prescriptionId }
   ```

2. **不要使用数字格式的医生ID**
   ```javascript
   // ❌ 错误
   doctor_id: '1'  // 数字！
   ```

3. **不要直接访问可能不存在的属性**
   ```javascript
   // ❌ 错误
   const reply = data.reply;  // 不安全
   ```

4. **不要频繁跳过检查**
   ```bash
   # ❌ 不要经常这样做
   git commit --no-verify
   ```
